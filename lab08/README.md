### Пуассоновский поток. События на сервере

Моделирование потока запросов на сервер.

Использовать модель простейшего потока событий, где событие — это поступление запроса на сервер.

- определить число запросов за интервал времени T (выбрать самостоятельно)
- построить эмпирическое распределение числа запросов за интервал времени T
- вычислить среднее и дисперсию числа запросов за интервал времени T
- сделать вывод

## Теория

### Простейший (Пуассоновский) поток событий

Простейший поток — это поток событий, удовлетворяющий трём условиям:

1. **Стационарность** — вероятность появления определённого числа событий на интервале зависит только от длины интервала, а не от его положения на оси времени.

2. **Отсутствие последействия** — число событий на непересекающихся интервалах — независимые случайные величины.

3. **Ординарность** — вероятность появления двух и более событий за бесконечно малый промежуток времени пренебрежимо мала.

### Экспоненциальное распределение интервалов

В простейшем потоке интервалы между соседними событиями (время ожидания следующего запроса) распределены **экспоненциально** с параметром λ:

```
F(t) = 1 - e^(-λt),  t ≥ 0
f(t) = λ · e^(-λt),  t ≥ 0
```

где λ — интенсивность потока (среднее число запросов в единицу времени).

**Свойства:**
- M[τ] = 1/λ — среднее время между запросами
- D[τ] = 1/λ² — дисперсия времени между запросами
- **Отсутствие памяти**: P(τ > s + t | τ > s) = P(τ > t) — время ожидания не зависит от того, сколько уже ждали

### Распределение Пуассона

Число событий N(T) за фиксированный интервал T в простейшем потоке имеет **распределение Пуассона** с параметром Λ = λT:

```
P(k) = (λT)^k · e^(-λT) / k!,  k = 0, 1, 2, ...
```

где Λ = λT — среднее число запросов за интервал T.

**Ключевые свойства:**

| Характеристика | Формула | Значение |
|----------------|---------|----------|
| Среднее | M[N(T)] | λT |
| Дисперсия | D[N(T)] | λT |
| Стандартное отклонение | σ[N(T)] | √(λT) |

Для распределения Пуассона **среднее равно дисперсии** — это характерный признак, который можно проверить на симуляции.

### Форма распределения

- При **λT < 10** — распределение скошено вправо (асимметрично)
- При **λT ≈ 10–20** — начинает напоминать колокол
- При **λT > 20** — приближается к нормальному распределению N(λT, λT) (центральная предельная теорема)

### Теорема Пуассона (закон больших чисел)

При увеличении числа экспериментов N эмпирические оценки среднего и дисперсии сходятся по вероятности к теоретическим значениям:

```
X̄ → λT,  S² → λT  при N → ∞
```

Это означает, что при достаточно большом N результаты моделирования должны хорошо совпадать с теорией.

---

## Как работает программа

1. **Моделирование**: Для каждого из N экспериментов:
   - Генерируются интервалы между запросами по формуле `τ = -ln(U)/λ`, где U ~ Uniform(0,1)
   - Считается, сколько запросов уложилось в интервал [0, T]

2. **Эмпирическое распределение**: Строится гистограмма частот числа запросов

3. **Сравнение**: Эмпирическое распределение накладывается на теоретическую формулу Пуассона

4. **Статистика**: Вычисляются эмпирические среднее и дисперсия, сравниваются с λT

---

## Выводы

После запуска симуляции программа автоматически формирует заключение о расхождении эмпирических и теоретических характеристик. Ожидается:

1. **Эмпирическое среднее** близко к λT (расхождение < 5% при N=1000)
2. **Эмпирическая дисперсия** близка к λT (расхождение < 10% при N=1000)
3. **Распределение** визуально совпадает с теоретическим
4. **Среднее ≈ дисперсия** — подтверждает, что поток является Пуассоновским

При увеличении N расхождение уменьшается (сходимость по закону больших чисел). При увеличении λT форма распределения становится более симметричной (приближение к нормальному).

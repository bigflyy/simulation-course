# Лабораторная работа 5. Моделирование случайных событий

## Задание

### 5.1. Приложение "Скажи да или нет"
Реализовать приложение, которое с заданной вероятностью $p$ отвечает "ДА" или "НЕТ" на вопрос пользователя.

### 5.2. Шар предсказаний (Magic 8-Ball)
Реализовать приложение-шар предсказаний, моделирующее выбор события из полной группы попарно несовместных событий с заданными вероятностями.

## Теоретические сведения

### Случайные объекты и базовый датчик

Случайные объекты:
1. **Случайные события**
2. Случайные величины
3. Случайные процессы
4. Случайные потоки событий

Базовый датчик (БД) генерирует $\alpha \in [0; 1]$. Через преобразование $\alpha$ получаем нужный случайный объект $X$.

### Геометрический смысл вероятности

На отрезке $[0; 1]$ случайное число $\alpha$ (из БД) попадает в подотрезок $[0; p)$ с вероятностью $p$:

$$P\{\alpha < p\} = p$$

### Датчик случайного события

Пусть $A$ — случайное событие с вероятностью $p = P\{A\}$.

**Алгоритм:**
1. Генерируем $\alpha$ из базового датчика
2. Если $\alpha < p$ — событие $A$ **наступило** (ДА)
3. Если $\alpha \geq p$ — событие $A$ **не наступило** (НЕТ)

### Генерация события из полной группы

Пусть $A_1, A_2, \ldots, A_m$ — **полная группа попарно несовместных событий**:

$$P\{A_i\} = p_i, \quad \sum_{i=1}^{m} p_i = 1$$

Отрезок $[0; 1]$ разбивается на интервалы длиной $p_1, p_2, \ldots, p_m$:

$$[0;\; p_1], \quad [p_1;\; p_1+p_2], \quad \ldots, \quad [p_1+\ldots+p_{m-1};\; 1]$$

**Алгоритм:**
1. $A := \alpha$ (из базового датчика), $k := 1$
2. $A := A - p_k$
3. Если $A \leq 0$, то произошло $A_k$; иначе $k := k + 1$, перейти к шагу 2

### Что делать, если в группе есть совместные события или группа не полна?

Необходимо преобразовать группу в полную группу попарно несовместных событий. Например, если даны события {«будет тепло», «пойдёт дождь»}, то полная группа:
- «будет тепло без дождя»
- «будет тепло и дождь»
- «будет холодно и дождь»
- «будет холодно без дождя»

### Статистическая обработка результатов

Для проверки корректности генерации проводится $N$ экспериментов и подсчитываются **относительные частоты** (эмпирические вероятности):

$$\hat{p}_k = \frac{n_k}{N}$$

где $n_k$ — число появлений события $A_k$, $N$ — общее число попыток (объём выборки).

При большом $N$ эмпирические частоты $\hat{p}_k$ должны быть близки к теоретическим вероятностям $p_k$.

## Вывод

Реализованы два приложения для моделирования случайных событий.

**Задание 5.1:** Датчик случайного события сравнивает случайное число $\alpha \sim U(0,1)$ с заданной вероятностью $p$. Метод основан на геометрическом смысле вероятности: доля отрезка $[0; p)$ в интервале $[0; 1)$ равна $p$.

**Задание 5.2:** Реализован алгоритм генерации события из полной группы несовместных событий. Алгоритм последовательно вычитает вероятности $p_k$ из случайного числа $\alpha$; событие $A_k$ считается наступившим, когда остаток становится $\leq 0$. Вероятности задаются пользователем через UI-поля, последняя вычисляется автоматически из условия нормировки ($\sum p_i = 1$, без процентов).
